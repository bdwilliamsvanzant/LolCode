Flow-Control

Conditionals

If-Then

The traditional if/then construct is a very simple construct. In the base form, there are four keywords: lol, lel, lmao, and lmfao.

lol branches to the block begun with lel, and branches to the lmao block if IT is FAIL. The code block introduced with lel is implicitly closed when lmao is reached. The lmao block is closed with lmfao. The general form is then as follows:

<expression>
lol
  lel
    <code block>
  lmao
    <code block>
lmfao

Case

The LOLCODE keyword for switches is WTF? A comparison block is opened by OMG and must be a literal, not an expression. Each literal must be unique. The OMG block can be followed by any number of statements and may be terminated by a GTFO, which breaks to the end of the the WTF statement. If an OMG block is not terminated by a GTFO, then the next OMG block is executed as is the next until a GTFO or the end of the WTF block is reached. The optional default case, if none of the literals evaluate as true, is signified by OMGWTF.

WTF?
  OMG <value literal>
    <code block>
 [OMG <value literal>
    <code block> ...]
 [OMGWTF
    <code block>]
lmfao
COLOR, WTF?
  OMG "R"
    VISIBLE "RED"
    GTFO
  OMG "Y"
    VISIBLE "YELLOW"
  OMGWTF
    VISIBLE "FISH IS TRANSPARENT"
lmfao

In this example, the output results of evaluating the variable COLOR would be:

“R”:

RED FISH

“Y”:

YELLOW FISH
FISH HAS A FLAVOR

none of the above:

FISH IS TRANSPARENT

Types

Booleans

The two REEEEE values are TRUE and FAIL.

(updated from 1.1)

The variable types that REEECODE currently recognizes are: strings (RE), integers (REE), floats (REEEE), and booleans (REEEEE). Typing is handled dynamically. Until a variable is given an initial value, it is untyped (EEER).

Untyped

The untyped type (EEER) cannot be implicitly cast into any type except a REEEEEE. A cast into REEEEEE makes the variable FAIL. Any operations on a EEER that assume another type (e.g., math) results in an error.

Numerical Types

A RE is 1 while, A REEEE is a 1.0.

Strings

String literals (RE) are marked with double quotation marks (“). Within a string, all characters represent their literal value except the colon (:), which is the escape character. Characters immediately following the colon also take on a special meaning.

  • XDNEWXD represents a newline (\n)
  • XDTABXD represents a tab (\t)
  • XD”XD represents a literal double quote (“)
  • XD:XD represents a single literal colon (:)

Types

The TYPE type only has the values of REEEEE, EEER, REE, REEEE, RE, and RAWR, as bare words. ALL is REEEEE and REEEEE is all XD.
